## Glossaire des termes 2026

| Terme | Définition |
|---|---|
| **AGENTS.md** | Fichier d'instructions tool-agnostique pour agents codeurs, standardisé par Linux Foundation Agentic AI Foundation (60K+ repos). |
| **Active CPU pricing** | Tarification Vercel Fluid Compute facturant uniquement le temps CPU actif (pas l'attente I/O). |
| **Agent (vs Workflow)** | LLM dirigeant dynamiquement son propre process, par opposition aux workflows orchestrés via paths de code prédéfinis. |
| **Agent Teams** | Mode Claude Code v2.1.32+ permettant à un lead session de spawn des teammates avec task list partagée et messaging peer-to-peer. |
| **AI Gateway (Vercel)** | Endpoint unique pour des centaines de modèles, avec routing, fallback, observabilité, et BYOK. |
| **AI SDK** | Vercel AI SDK ; v6 depuis décembre 2025. Architecture Core / UI / RSC (paused) / Gateway / DevTools / MCP. |
| **AI SDK RSC** | Module pour la generative UI ; *en pause* en 2026. À ne pas adopter pour de nouveaux projets. |
| **BYOK** | Bring Your Own Key. Pattern où chaque tenant fournit ses propres clés provider (chiffrées en envelope encryption). |
| **Cache Components** | Modèle de caching Next.js 16 où tout est dynamique par défaut, on opte explicitement avec `'use cache'`. |
| **Claude Agent SDK** | SDK Anthropic qui power Claude Code, exposé en `@anthropic-ai/claude-agent-sdk` ; design center file-system, long-running. |
| **Compound correctness** | Capacité d'un agent de coding à compounder la correctness sur des tâches longues plutôt que d'erreurs propager. Atteint avec Claude 3.5 Sonnet (oct. 2024). |
| **Context engineering** | Discipline de gérer la fenêtre de contexte comme une ressource finie (vs. *prompt engineering*). |
| **Context rot** | Dégradation du recall avec la croissance des tokens, à cause de l'attention n². |
| **Contextual Retrieval** | Technique Anthropic d'augmentation des chunks avec contexte généré par Claude avant embedding. −67 % failure rate sur top-20 (combiné). |
| **DAL (Data Access Layer)** | Module qui possède auth + access control ; les Server Components ne voient que des DTOs. Pattern Vercel. |
| **Dark factory** | Org où le code n'est ni écrit ni reviewé par humains. Réel à StrongDM, Spotify (depuis déc. 2025). |
| **EARS notation** | When/While/Where syntax for acceptance criteria, used in Kiro specs. |
| **Eval-driven dev (EDD)** | Pratique de baser le dev sur les évals, comme TDD se basait sur les tests. La compétence rare en 2026. |
| **Evaluator-optimizer** | Pattern workflow generator/critic loop. |
| **Fluid Compute** | Plateforme Vercel par défaut depuis avril 2025 ; remplace serverless classique avec in-function concurrency. |
| **G-Eval** | Pattern d'eval LLM-as-judge avec chain-of-thought + form-filling (Eugene Yan). |
| **Hooks (Claude Code)** | Mécanisme déterministe d'extension : commands shell, HTTP endpoints, MCP tools, prompts Claude. Le seul *enforced* mechanism. |
| **Indirect prompt injection** | Injection via tool results (web pages, emails, retrieved docs). A dépassé l'injection directe en 2025. |
| **MCP (Model Context Protocol)** | Protocole standardisé pour les tools / resources d'agents ; spec OAuth 2.1 mars 2026. |
| **Mutation gate** | Gate CI qui requiert que les tests killent les mutants générés (Stryker, mutmut, PIT). Vrai signal vs coverage. |
| **needsApproval** | Flag AI SDK 6 sur un tool : bloque l'exécution jusqu'à approbation user. |
| **Orchestrator-worker** | Pattern où un LLM central décompose dynamiquement, les subtasks ne sont pas prédéfinies. |
| **PPR (Partial Prerendering)** | Technique de rendu hybride statique/dynamique ; en 2026, c'est un détail d'implémentation sous Cache Components. |
| **Prompt caching** | Mécanisme provider qui cache un prefix prompt. 90 % off Anthropic, 50 % off OpenAI. |
| **proxy.ts** | Renamed `middleware.ts` en Next.js 16. Réseau, runtime Node, à ne *jamais* utiliser pour l'authz seule. |
| **PuppeteerVerification** | Pattern Anthropic harness où l'agent verify end-to-end avec Puppeteer avant de marquer une feature done. |
| **Resumable streams** | Streams SSE Redis-backed permettant la reprise après refresh / disconnect. AI SDK 5+. |
| **RRF (Reciprocal Rank Fusion)** | Méthode de fusion de listes rankées pour hybrid retrieval (dense + BM25). |
| **Server Action** | Fonction server-side appelée depuis un client React, marked `'use server'`. À traiter comme un endpoint POST public. |
| **Skills (Anthropic)** | Knowledge progressively-disclosed (`SKILL.md` + frontmatter) chargée à la pertinence. Évite de bloater CLAUDE.md. |
| **Spec-Driven Development (SDD)** | Pratique de spec-first augmentée par AI tooling. GitHub Spec Kit, AWS Kiro. RFC/PRD rebrandé pour l'ère IA. |
| **stopWhen** | Option AI SDK 5+ pour contrôler la fin d'une boucle agent (`stepCountIs(N)`, `hasToolCall(...)`). |
| **TAG (Test-After-Generation)** | Pattern où l'agent génère code + tests en parallèle ; les tests sont gated par mutation score. |
| **ToolLoopAgent** | Classe AI SDK 6 pour réutiliser un agent à travers plusieurs call sites. |
| **Trust boundary** | Frontière entre contenu trusted (system prompt) et untrusted (tool results, retrieved). À tagger explicitement. |
| **Turbopack** | Bundler Next.js 16 stable. Successeur Webpack pour dev et prod. |
| **Vercel Sandbox** | Compute primitive Vercel pour exécuter du code non-trusté (output d'agent, scripts user). Microvms Firecracker. |
| **View Transitions** | API React 19.2 pour les transitions DOM ; route changes morphing dans Next.js 16. |
| **Workflow (vs Agent)** | LLMs orchestrés via paths de code prédéfinis. À préférer à *agent* tant que la tâche se modèle. |
